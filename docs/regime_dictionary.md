# Aggregation Regime Dictionary
**Repository:** ethical-alpha-audit-paper-5-sensitivity-study
**Version:** v2.0
**Status:** This document defines every aggregation regime referenced by Paper 5
manuscript and supplementary materials. Each numerical claim in the paper is
labelled with one regime ID; this dictionary maps that ID to its scope and
source-file provenance.

| Regime ID | Aggregation scope | Source file(s) | Used in main paper |
|---|---|---|---|
| AR1 (threshold-profile regime) | Per-profile aggregation across 5 scenario families × 5 replicates | `outputs/tables/table1_thresholds.csv` | Table 1 |
| AR2 (architecture-comparison regime) | Paired non-compensatory vs weighted-average-composite runs at 20% unsafe base rate, 5 replicates | `outputs/raw/compensatory_comparison.csv` | Table 2; Architecture comparison subsection |
| AR3 (inference regime) | Bootstrap mean across the full evaluation matrix (25 evaluations of moderate non-compensatory profile across 5 scenarios × 5 replicates) | `outputs/processed/inference_results.json` | Lifecycle and inference subsection |
| AR_lifecycle | Lifecycle module aggregation across 12 simulated time periods | `outputs/processed/lifecycle_summary_v2.json`, `outputs/processed/lifecycle_per_period.csv` | Lifecycle and inference subsection |
| AR_replication | Single-seed (seed = 42) replication build comparison | `reproducibility/replication_report.json` | Lifecycle and inference subsection |
| AR_nsga2 | NSGA-II multi-seed search (seeds 42, 123, 7777; population 60, 30 generations, 1,800 evaluations per seed) | `outputs/processed/pareto_solutions.csv`, `outputs/processed/nsga2_convergence.json`, `outputs/processed/pareto_seed_stability.csv` | Pareto frontier subsection; NSGA-II convergence supplement |
| AR_sobol_N1024 | Sobol global sensitivity decomposition at N = 1024 base samples (9,216 total evaluations) with 50-resample bootstrap CIs | `outputs/processed/sobol_convergence.json`, `outputs/tables/sobol_*.csv` | Sensitivity analysis subsection |

## Provenance and reproducibility
Every numerical value cited in the manuscript or supplementary materials is
generated by the simulation pipeline (entry point: `reproduce_all.py`) under
seed 42 and the configuration registered in `config/parameters.yaml` and
`config/harness_settings.json`. The Stage 7 orchestrator (`stage7_orchestrator.py`
and `run_sobol_1024_min.py`) regenerates all artefacts listed above. After
regeneration, hash-chain validation is executed via `scripts/hash_manifest.py`
followed by `scripts/validate_outputs.py`.

## Forbidden cross-regime claims
Numbers from one regime must not be mixed with numbers from another in a single
claim without explicit regime annotation. Specifically:
- The throughput value 0.213 (AR1, moderate row) is not the same as 0.218 (AR2 NC mean).
- The detection rate 0.999 (AR1, moderate) is not the same as 0.998 (AR2 NC mean).
- The false-negative-harm value reported in Table 1 (AR1) is not the same as
  the value reported in Table 2 (AR2).

These differences arise from differing aggregation scopes, not from instability.
